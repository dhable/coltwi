Building a release
-----------------------------------
1. Update the version number in the build.sbt file
2. Edit source/other/coltwi, src/other/coltwi.cmd to
   update the name of the coltwi jar file to reflect the
   new version number.
3. Reload the sbt configuration.
4. In sbt, use the `stage` command to build and stage the files to
   target/coltwi-<ver>
5. Run just package <vers>
   - Removes and .DS_Store files in the staged directories.
   - Creates zip file of target/coltwi-<vers> folder
   - Copies zip file to ~/Dropbox/coltwi
6. Test that new package works
7. Update the download link in the README.md file
